## ⚠️ အရေးကြီးသော ဥပဒေရေးရာ ထုတ်ပြန်ချက်

**ဤစနစ်သည် ပရိုဂရမ်ရေးသားခြင်း သင်ယူရေးအတွက် နမူနာကုဒ်သာဖြစ်ပါသည်**

ဤ repository တွင် Next.js နှင့် Supabase အသုံးပြု၍ အချက်အလက်စစ်ဆေးခြင်း လုပ်ငန်းစဉ်ကို တည်ဆောက်ပုံ သရုပ်ပြသော **ပညာရေးရည်ရွယ်ချက်အတွက် နမူနာကုဒ်များ** ပါဝင်ပါသည်။

### ဤစနစ်သည် အောက်ပါအရာများ မဟုတ်ပါ

❌ မည်သည့် အစိုးရ သို့မဟုတ် အဖွဲ့အစည်း၏ တရားဝင်စနစ် မဟုတ်ပါ  
❌ လက်တွေ့အသုံးပြုရန် မသင့်လျော်ပါ  
❌ မည်သည့် အဖွဲ့အစည်းနှင့်မှ ဆက်စပ်မှု မရှိပါ  
❌ အမှန်တကယ် စစ်ဆေးနိုင်သော ဝန်ဆောင်မှု မဟုတ်ပါ  
❌ ဥပဒေရေးရာ အကြံဉာဏ် သို့မဟုတ် အသိအမှတ်ပြုလက်မှတ် မဟုတ်ပါ

### ဤစနစ်သည် အောက်ပါအရာများ ဖြစ်ပါသည်

✅ ပညာရေး ကုဒ်နမူနာ  
✅ နည်းပညာဆိုင်ရာ ဗိသုကာ သရုပ်ပြခြင်း  
✅ အများသုံး သင်ယူရေး အရင်းအမြစ်  
✅ ပရိုဂရမ်ရေးသားခြင်း လေ့ကျင့်ရေး ပရောဂျက်

### အချက်အလက်ဆိုင်ရာ ထုတ်ပြန်ချက်

- နမူနာအချက်အလက်အားလုံးသည် **စိတ်ကူးယဉ်** ဖြစ်သည်
- field အမည်များအားလုံးသည် **ယေဘူယျ နမူနာများ** ဖြစ်သည်
- UI စာသားအားလုံးသည် **သရုပ်ပြရန်သာ** ဖြစ်သည်
- မည်သည့် လူအမှန်၊ စာရွက်စာတမ်း သို့မဟုတ် အဖွဲ့အစည်းကိုမှ ကိုယ်စားပြုခြင်း မရှိပါ

### ဥပဒေရေးရာ သတိပေးချက်

**အာမခံချက် မရှိပါ**: ဤဆော့ဖ်ဝဲကို မည်သည့်အာမခံချက်မျှမပါဘဲ "ရှိသည့်အတိုင်း" ပေးအပ်ထားပါသည်။

**တာဝန်ယူမှု မရှိပါ**: ရေးသားသူများသည် မည်သည့် အသုံးပြုမှု၊ လွဲမှားစွာအသုံးပြုမှု သို့မဟုတ် အကျိုးဆက်များအတွက် တာဝန်မယူပါ။

**တားမြစ်ထားသော အသုံးပြုမှုများ**: လိမ်လည်မှု၊ အတုအယောင်ပြုလုပ်ခြင်း၊ ပုဂ္ဂိုလ်ကျူးလွန်မှု သို့မဟုတ် အများပြည်သူကို လှည့်ဖြားခြင်းအတွက် အသုံးမပြုပါနှင့်။

**သင်၏တာဝန်**: ဤကုဒ်ကို မည်သို့ပင်အသုံးပြုသည်ဖြစ်စေ သင်သည် အပြည့်အဝတာဝန်ယူရမည်။

### Database တည်ဆောက်ခြင်း

Supabase project တစ်ခုဖန်တီး၍ ဤ SQL ကို run ပါ:

```sql
-- သင်ယူရန်အတွက် နမူနာ table (အမှန်တကယ် အချက်အလက် ဖွဲ့စည်းပုံ မဟုတ်ပါ)
CREATE TABLE demo_documents (
  idx SERIAL PRIMARY KEY,
  document_no VARCHAR(50) UNIQUE NOT NULL,
  issue_year VARCHAR(20),
  reference_no VARCHAR(50),
  holder_name VARCHAR(200),
  issue_date VARCHAR(50),
  parent_name_1 VARCHAR(200),
  parent_name_2 VARCHAR(200),
  category VARCHAR(50),
  remarks TEXT,
  status VARCHAR(20) DEFAULT 'active',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  uuid UUID DEFAULT gen_random_uuid() UNIQUE
);

CREATE INDEX idx_document_no ON demo_documents(document_no);
CREATE INDEX idx_uuid ON demo_documents(uuid);

-- စိတ်ကူးယဉ် စမ်းသပ်အချက်အလက် ထည့်သွင်းပါ
```

## 📖 သင်ယူရေး အကြောင်းအရာများ

### လုံခြုံရေး ပုံစံများ

- **Input Validation**: UUID နှင့် ဂဏန်း format စစ်ဆေးခြင်း
- **CAPTCHA**: စိတ်ကြိုက် pixel-based စစ်မှန်းခြင်း
- **SQL Injection ကာကွယ်ခြင်း**: တင်းကျပ်သော format စစ်ဆေးခြင်း
- **Error Handling**: ပေါင်းစည်းထားသော error စာမျက်နှာများ

### ကုဒ်နမူနာများ

**UUID Validation**
```typescript
const isUUID = (v: string) =>
  /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i.test(v)
```

**စာရွက်စာတမ်းနံပါတ် Validation**
```typescript
if (!/^\d+$/.test(id)) {
  redirect("/verify/not-found")
}
```

**ဘာသာစကားများစွာ နံပါတ်ပြောင်းလဲခြင်း**
```typescript
function toMyanmarNumber(input: string): string {
  const map: Record<string, string> = {
    "0": "၀", "1": "၁", "2": "၂", "3": "၃", "4": "၄",
    "5": "၅", "6": "၆", "7": "၇", "8": "၈", "9": "၉",
  }
  return input.replace(/[0-9]/g, (d) => map[d])
}
```

5. **အပြည့်အဝတာဝန်ယူပါ** သင်၏ version အတွက်
